# ■ 基本設計書：Todoアプリ（ログイン機能なし）

## 1. システム概要

本システムは、ユーザーがタスク（Todo）を登録・更新・削除できるシンプルなWebアプリケーションである。
データはローカルストレージ（LocalStorage）に保存し、サーバーは使用しない。

---

## 2. 想定ユーザー

- 個人利用ユーザー
- ブラウザ上でタスク管理したいユーザー

---

## 3. 画面構成

### 3.1 メイン画面（Todo一覧画面）

#### 目的

Todoの追加・一覧表示・完了・削除を行う

#### UI構成

- 入力エリア
  - テキストボックス（Todo入力）
  - 追加ボタン

- Todo一覧
  - チェックボックス（完了/未完了）
  - Todoタイトル
  - 削除ボタン

---

## 4. 機能一覧

### 4.1 Todo追加機能

- 入力フォームからTodoを追加
- 空文字は登録不可
- 登録時にIDを自動付与

---

### 4.2 Todo一覧表示機能

- LocalStorageから取得して一覧表示
- 最新追加は下に追加（または上に追加）

---

### 4.3 完了状態切り替え機能

- チェックボックスで完了/未完了を切替
- 完了時は打ち消し線表示

---

### 4.4 削除機能

- 各Todo単位で削除ボタン
- 即時LocalStorageから削除

---

## 5. データ設計

### 5.1 Todoデータ構造

```js
Todo {
  id: string,        // ユニークID（Date.now()など）
  title: string,     // タスク内容
  completed: boolean,// 完了状態
  createdAt: number  // 作成日時
}
```

---

### 5.2 保存方法

- LocalStorageキー：`todos`

```js
localStorage.setItem("todos", JSON.stringify(todoList));
```

---

## 6. 画面遷移

単一画面アプリ（SPAではあるがルーティングなし）

```
メイン画面のみ
```

---

## 7. 処理フロー

### 7.1 追加処理

1. 入力値取得
2. バリデーション（空チェック）
3. Todoオブジェクト作成
4. 配列に追加
5. LocalStorage更新
6. 画面再描画

---

### 7.2 表示処理

1. LocalStorageから取得
2. JSONパース
3. DOM生成
4. 画面描画

---

### 7.3 削除処理

1. ID取得
2. 配列から除外
3. LocalStorage更新
4. 再描画

---

### 7.4 完了切替処理

1. ID取得
2. 該当Todoを検索
3. completedを反転
4. LocalStorage更新
5. 再描画

---

## 8. 非機能要件

### 8.1 パフォーマンス

- 100件程度のTodoを想定し軽量動作

### 8.2 対応ブラウザ

- Chrome最新版
- Edge最新版
- Safari最新版

### 8.3 データ永続化

- LocalStorage依存（サーバーなし）
